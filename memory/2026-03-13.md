### Debugging manhog seller-volumes chart — 21:22 UTC
- **Root cause**: x402scan only has materialized views for `7d`, `30d`, and `0d` (all-time). The manhog "72 hours" view was passing `timeframe: 3`, which tried to query `recipient_stats_bucketed_3d` — doesn't exist → 500 error.
- **Fix**: Map all short timeframes to `7` (7d MV). The MV suffix logic lives in `apps/scan/src/lib/time-range.ts` → `getMaterializedViewSuffix()` which just does `${timeframe}d`.
- **Key lesson**: x402scan's bucketed tRPC endpoint returns 500 (not empty) when the MV doesn't exist. The original `trpcFetch` was treating 500 as an error but the `catch` block in `getStablesBucketedStats` was swallowing it and returning `[]`. Added verbose logging to diagnose.
- **StableStudio domain**: `stablestudio.io` (NOT `.dev`) — the resources are registered under `.io` in x402scan.
