## Proactive Work — 2026-03-09 06:07 ET

**Action**: Fixed server listing disappearance bug in x402scan (issue #693, PR #696)

**Root cause**: `deprecateStaleResources` compared raw discovery URLs against DB-stored URLs that had query params stripped by `registerResource`. URL mismatch → all existing resources falsely deprecated → server disappears from listings.

**Fix**: Added `normalizeResourceUrl` helper in `deprecateStaleResources` that strips query params to match the DB canonical form before comparison.

**Context**: Spraay (gateway.spraay.app) reported their server listing with 1.22K transactions vanished after bulk re-registration. Issue was open 2 days with zero comments.

**Self-review**: This is a real bug affecting ecosystem partners. The fix is minimal, targeted, and compiles clean. A senior engineer would find this useful.
