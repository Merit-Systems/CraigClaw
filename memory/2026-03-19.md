## agentcash redeem investigation — 2026-03-19 ~19:15 UTC

- Ryan reported invite codes created on manhog (x402scan) 404-ing when redeemed in agentcash
- GET `/api/redeem` has `code.startsWith('MRT')` gate — only proxies to x402scan for MRT-prefixed codes
- POST `/api/redeem` correctly falls through to x402scan for ANY code not found locally
- Created PR #335 to remove the MRT prefix gate — Ryan said delete it, closed + deleted branch
- Root cause actually turned out to be **bun caching**: Poncho runs `bun x agentcash@latest server` but bun caches the `@latest` resolution, so Poncho was stuck on agentcash 0.9.5 while latest is 0.12.0
- The old 0.9.5 version likely has different/missing redeem logic
- `X402_FALLBACK` URL uses `x402scan.com` (no www) which 307-redirects to `www.x402scan.com` — Node fetch follows it fine but it's an unnecessary redirect
